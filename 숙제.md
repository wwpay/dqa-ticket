## 2. 확정된 결정사항 (되돌리지 말 것)

- **편집 잠금 구조**: detail 페이지 진입 시 `lockTicket(rowId)` 호출. 이미 잠긴 경우 읽기전용 배너 표시 + 폼 비활성화. 저장/취소/삭제/beforeunload 시 `unlockTicket` 호출. 잠금 만료: 30분.
  - GAS: `COL.LOCKED_AT = 16` (Q열), `lockTicket` / `unlockTicket` / `checkLock`
  - api.js: 같은 이름의 wrapper 함수 (callGAS 사용)
  - detail.js: `currentRowId`, `isReadOnly` 모듈 변수. `enterReadOnlyMode()` / `handleLockBeforeUnload()`
  - index.js: `ticket.locked_at` 30분 이내 시 🔒 아이콘 (buildRow)
- **DnD 실시순서**: 제거 완료 (cc58900). 재도입 시 아래 TODO #2 규칙 준수
- **헤더 필터 구조**: `wrap()` 함수 → `.th-content > (.th-row > .th-filter-wrap + 아이콘) + .th-filter-badge`
  - 컬럼명은 항상 유지, 활성 필터는 하단 뱃지(× 포함)로 분리
  - `×` 클릭 이벤트는 `th-filter-clear` 클래스 기반 위임
- **그룹 헤더 색상**: `--*-mid` 변수 (ww-mid/#dbeafe, mvn-mid/#dcfce7, done-mid/#e5e7eb, hold-mid/#d1d5db)
- **섹션별 th 색상**: `.section-X .ticket-table th` 오버라이드 (bg=*-light, color=섹션 accent)
- **보류 & N/A**: 슬래시 `/` → 앰퍼샌드 `&` (i18n 4개 언어 모두)
- **priority 필드**: detail 페이지에서 readonly input (DnD로만 변경)
- **btn-wjira-link**: 완전 제거
- **ticket-id-wrap**: `flex: 1` (신규 등록 메타 3단 균등 유지)
